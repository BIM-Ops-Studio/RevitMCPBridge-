# Construction Administration

## CA Phase Overview

### Architect's Responsibilities
| Task | Description |
|------|-------------|
| Submittals | Review shop drawings, samples |
| RFIs | Respond to requests for information |
| Site visits | Periodic observation |
| Field reports | Document observations |
| Change orders | Review and recommend |
| Pay applications | Review and certify |
| Punch list | Prepare and verify |
| Closeout | Review final documentation |

### Standard of Care
| Responsibility | Level |
|----------------|-------|
| Design conformance | Observe, not inspect |
| Construction means | Contractor's responsibility |
| Safety | Contractor's responsibility |
| Schedule | Contractor's responsibility |
| Quality | Contractor's responsibility |

## Submittals

### Types of Submittals
| Type | Purpose |
|------|---------|
| Shop drawings | Fabrication details |
| Product data | Manufacturer literature |
| Samples | Physical samples |
| Mock-ups | Full-scale assemblies |
| Certificates | Compliance documentation |
| Test reports | Performance verification |

### Review Actions
| Action | Meaning |
|--------|---------|
| Approved | Proceed with work |
| Approved as noted | Incorporate notes, proceed |
| Revise and resubmit | Correct, return for review |
| Rejected | Do not proceed |
| Reviewed | No exceptions taken |
| Not reviewed | Out of scope |

### Review Timeline
| Phase | Duration |
|-------|----------|
| Initial review | 10-14 days typical |
| Resubmittal | 7-10 days |
| Expedited | Per agreement |
| Approval stamp | Before work begins |

## RFIs (Requests for Information)

### Purpose
| Use | Examples |
|-----|----------|
| Clarification | Conflicting dimensions |
| Interpretation | Ambiguous specifications |
| Missing information | Incomplete details |
| Suggested change | Alternate approach |

### RFI Process
```
1. Contractor identifies issue
2. Contractor submits RFI
3. Architect reviews
4. Architect responds in writing
5. Contractor proceeds (or appeals)
6. Document in project record
```

### Response Time
| Priority | Response |
|----------|----------|
| Urgent | 1-2 days |
| Standard | 5-7 days |
| Complex | 10-14 days |
| Research required | Per discussion |

### RFI Log Contents
| Field | Description |
|-------|-------------|
| RFI number | Sequential |
| Date received | Contractor submission |
| Subject | Brief description |
| From | Submitting party |
| To | Responding party |
| Date due | Response deadline |
| Date answered | Actual response date |
| Status | Open/closed |

## Change Orders

### Types of Changes
| Type | Description |
|------|-------------|
| Change order | Formal modification |
| Construction change directive | Proceed before price |
| Field order | Minor changes |
| Clarification | No cost impact |

### Change Order Process
```
1. Need for change identified
2. Contractor provides proposal
3. Architect reviews scope and cost
4. Owner approves/negotiates
5. Execute change order
6. Contractor proceeds
7. Document in record
```

### Pricing Methods
| Method | Use |
|--------|-----|
| Lump sum | Defined scope |
| Unit price | Quantity-based |
| Time and materials | Undefined scope |
| Cost plus | Emergency work |

### Standard Markups
| Cost Component | Markup |
|----------------|--------|
| Labor | 15-25% |
| Materials | 10-15% |
| Equipment | 10-15% |
| Subcontractor | 5-10% |
| Bond | 1-2% (if required) |
| Overhead | 10-15% |
| Profit | 5-10% |

## Pay Applications

### Review Process
```
1. Contractor submits application
2. Subcontractor waivers attached
3. Schedule of values updated
4. Stored materials documented
5. Architect reviews quantities
6. Architect certifies amount
7. Owner processes payment
```

### Schedule of Values
| Requirements | Description |
|--------------|-------------|
| Breakdown | By specification section |
| Detail | Reasonable line items |
| Basis | Installed value |
| Approval | Before first application |
| Updates | As work progresses |

### Retainage
| Phase | Typical Retainage |
|-------|-------------------|
| Progress payments | 5-10% |
| 50% complete | May reduce to 5% |
| Substantial completion | Release 50% |
| Final completion | Release remainder |

### Common Issues
| Issue | Resolution |
|-------|------------|
| Overbilling | Adjust quantities |
| Stored materials | Verify and document |
| Missing waivers | Withhold until received |
| Disputed work | Note in certification |

## Punch List

### Process
```
1. Contractor requests inspection
2. Architect tours with contractor
3. Architect documents deficiencies
4. Contractor corrects items
5. Architect reinspects
6. Repeat until complete
7. Issue certificate
```

### Punch List Item Types
| Type | Examples |
|------|----------|
| Incomplete | Missing hardware |
| Damaged | Scratched finish |
| Incorrect | Wrong color |
| Non-conforming | Wrong product |
| Functional | Doesn't operate |
| Aesthetic | Touch-up needed |

### Substantial Completion
| Criteria | Description |
|----------|-------------|
| Occupancy | Space is usable |
| Systems | All operational |
| Safety | All features complete |
| Remaining | Punch list only |
| Certificate | Issued by architect |
